# Action Capability Modeling

## When to use
Quando a tarefa envolver naming, schemas, actions, capabilities ou contract design.

## Goal
Formalizar intenção do produto de modo estável, extensível e audível.

## Inputs required
domínio atual, superfícies de controle atuais, Mark 1 e Mark 2 docs

## Output expectations
schemas, DTOs, capability metadata, mapeamentos

## Rules
toda capability relevante precisa ter contrato claro

## Anti-patterns
deixar capability implícita em scripts

## Validation
schema tests, revisão de cobertura de ações
