# QA Terminal Validation

## When to use
Quando a tarefa envolver validação final, acceptance, runbooks ou troubleshooting.

## Goal
Fechar o ciclo com prova terminal e documentação operacional completa.

## Inputs required
checklists do Mark 1, runbooks, validators existentes

## Output expectations
suíte final, acceptance matrix, troubleshooting, report templates

## Rules
sem validação terminal não há conclusão

## Anti-patterns
encerrar o ciclo sem prova integrada

## Validation
execução da suíte final do Mark 1
