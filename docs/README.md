# Docs

Organizacao da documentacao do projeto.

## Essenciais

Os documentos abaixo ficam na raiz de `docs/` porque definem escopo, arquitetura, padroes e fluxo geral do projeto:

- `PROJECT-SCOPE.md`
- `SIMULATION-ARCHITECTURE.md`
- `PROJECT-ARCHITECTURE.md`
- `MONOREPO-STRUCTURE.md`
- `AGENTS-AND-SKILLS.md`
- `DEVELOPMENT-STANDARDS.md`
- `TESTING-AND-FAILURE-MODEL.md`
- `CHECKLIST-FRAMEWORK.md`
- `PROJECT-EXECUTION-CHECKLIST.md`

## Decisions

`docs/decisions/` concentra escolhas oficiais, criterios de migracao e decisoes abertas ou fechadas.

## Contracts

`docs/contracts/` concentra contratos tecnicos por fase e especificacoes executaveis da stack.

## Runbooks

`docs/runbooks/` concentra ordem operacional, troubleshooting e guias de execucao.

## Reference

`docs/reference/` concentra material de consulta, inventarios e notas auxiliares que nao fazem parte do escopo geral obrigatorio.
