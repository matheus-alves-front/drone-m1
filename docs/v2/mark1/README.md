# Mark 1

## Nome
Drone Control Platform

## Missão do Mark 1
Transformar o projeto atual em uma plataforma controladora unificada do drone, ainda simulation-first, porém:
- operável por interface humana central
- operável por interface programática central
- pronta para crescer sem retrabalho estrutural

## Estado atual

O Mark 1 está fechado no repositório com:
- control plane explícito
- operator console funcional
- missão, safety, veículo, cenários e percepção expostos por superfícies unificadas
- read model separado de command plane
- validação final consolidada em `R15`

## O que NÃO é o Mark 1
- não é um MVP simplificado
- não é apenas um dashboard melhor
- não é apenas um wrapper de scripts
- não é o Mark 2 inteiro
