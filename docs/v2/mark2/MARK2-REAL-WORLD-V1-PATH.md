# Mark 2 Real-World V1 Path

## Objetivo
Manter a janela aberta para uma futura V1 física sem prometer operação real nesta rodada.

## Estratégia
O projeto continua simulation-first, mas a arquitetura já deve reduzir o custo de uma futura transição.

## O que a futura V1 real exigirá
- drivers físicos reais
- validação de runtime em hardware
- safety hardening adicional
- integração energética e térmica
- calibração
- operação supervisionada
- compliance e operação responsável

## O que o software já deve entregar antes disso
- actions e capabilities estáveis
- control plane maduro
- read model auditável
- payload/actuator abstractions previstas
- environment strategy clara
- runbooks maduros
- QA terminal forte

## Regra
A futura V1 real não deve exigir reescrever o sistema.  
Ela deve exigir principalmente:
- adapters
- hardening
- testes
- validação operacional
