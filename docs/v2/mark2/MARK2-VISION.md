# Mark 2 Vision

## Visão
Evoluir a Drone Control Platform para uma **Advanced Robotics Platform**, onde o drone é apenas a primeira implementação de referência.

## O que a plataforma deve suportar no futuro

- drone como veículo aéreo
- payloads diversos
- speaker/display/avatar
- câmeras e sensores adicionais
- futuros braços robóticos
- tool heads e actuators especializados
- múltiplos veículos e runtimes
- uso por operadores humanos
- uso por agentes de IA via API/MCP

## Filosofia
Não é um "app de drone".  
É um **hub de robótica avançada simulation-first, hardware-ready e capability-driven**.

## Tipos de expansão previstos

### Expansão de veículo
- aerial vehicle
- future ground base
- future stationary robotic cell

### Expansão de payload
- camera payload
- display payload
- audio payload
- delivery payload
- robotic arm payload

### Expansão de capability
- delivery planning
- inspection
- conversational guidance
- manipulation
- cooperative tasks
- autonomy recipes

### Expansão de interface
- operator UI
- machine API
- MCP tools
- workflow orchestration

## Resultado esperado do Mark 2
A plataforma passa a ser um runtime modular de robótica, não apenas um stack de drone.
