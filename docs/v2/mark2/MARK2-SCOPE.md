# Mark 2 Scope

## Objetivo
Planejar com precisão a próxima evolução do sistema para uma plataforma robótica modular.

## Entram no escopo do Mark 2 como planejamento
- capability registry
- plugin architecture
- payload abstraction
- actuator abstraction
- vehicle family abstraction
- action capability negotiation
- MCP-ready machine surface
- hardware readiness path
- future real-world deployment strategy

## Não entra nesta rodada como implementação
- braço robótico funcional completo
- múltiplos braços cooperativos completos
- operação real de limpeza/fachada/manipulação
- certificação real
- integração física final de payloads complexos

## O que o Mark 2 exige do Mark 1
- nomes e entidades generalizáveis
- ação formalizada
- capability discovery inicial
- separation of concerns forte
- control plane central
- UI desacoplada de runtime interno
